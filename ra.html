Producto

Lo mínimo necesario para que sea útil desde el día uno

No es un dashboard bonito sin función. La idea es administrar, consultar, comparar y notificar.

01

Agregar causas

Registro por competencia, tribunal, tipo, rol y año para dejar cada causa lista para seguimiento.

02

Pausar o activar

Control operativo simple: activa lo importante, pausa lo que no quieres revisar y elimina sin perder trazabilidad.

03

Historial de revisiones

Cada consulta queda registrada con fecha, estado, resultado normalizado y detección de cambios.

04

Correo automático

El usuario decide si quiere recibir avisos inmediatos o resúmenes periódicos por email.

Flujo

La web administra. El worker vigila.

Ese punto es clave. La página no debería depender de consultar en vivo cada vez que el usuario abre el dashboard.

1

Usuario configura

Agrega causas, define estado, revisa historial y activa o desactiva notificaciones.

2

Worker revisa

Un proceso separado consulta las fuentes, normaliza resultados y compara contra la revisión anterior.

3

Sistema notifica

Si hay cambios y el correo está activo, RA registra el evento y envía el aviso correspondiente.

Nota técnica importante

RA debe operar con una base de datos propia. El dashboard lee datos guardados por tu sistema; el proceso automático de revisión corre por separado. Esa separación evita una plataforma lenta, frágil y dependiente de consultas en vivo cada vez que alguien entra.
